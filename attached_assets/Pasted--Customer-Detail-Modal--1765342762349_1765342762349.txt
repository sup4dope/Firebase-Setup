이미지 인식에 한계가 있어서, 내가 원하는 **"고객 상세 모달(Customer Detail Modal)"**의 정확한 설계도를 텍스트로 다시 짰어.
기존의 다크모드 디자인 스타일은 유지하되, **반드시 아래 정의된 4개 섹션 구조와 필드 배치**를 엄격하게 지켜줘.

### 1. 전체 레이아웃 (Grid Layout)
모달 전체를 **좌우 2분할**하고, 우측은 다시 **상하 2분할**하는 구조야.

```text
[ 전체 화면 모달 (90vw x 90vh) ]
-----------------------------------------------------------------------
| (섹션 1) 좌측 패널        | (섹션 2) 우측 상단 패널                 |
| 고객/사업자 입력 폼       | 파일 리스트 및 문서 뷰어 (PDF/IMG)      |
| (너비: 40%, 높이: 100%)   | (너비: 60%, 높이: 50%)                  |
| 스크롤 가능               |                                         |
|                           |-----------------------------------------|
|                           | (섹션 3) 우하단 좌측  | (섹션 4) 우하단 우측 |
|                           | 상담 메모 (Chat UI)   | AI 질의 (Chat UI)    |
|                           | (너비: 30%, 높이: 50%)| (너비: 30%, 높이: 50%)|
-----------------------------------------------------------------------
2. [섹션 1] 좌측: 상세 정보 입력 (Input Form)
디자인: 각 그룹(2번, 3번)은 테두리(Border)가 있는 박스로 묶어서 구분해줘.

1. 유입경로 (최상단 단독)

[드롭다운] (옵션: 광고랜딩명, 외주, 고객소개)

2. 고객 정보 그룹 (Border Box)

Row 2-1 (한 줄에 배치): [이름] [신용점수] [주민등록번호(앞6)] - [주민등록번호(뒤1+마스킹)]

Row 2-2 (한 줄에 배치): [연락처(010)]-[중간4자리]-[끝4자리] [통신사(드롭다운: SKT, KT, LG, 알뜰폰...)]

Row 2-3 (한 줄에 배치): [자택주소(도로명 검색버튼 포함)] [상세주소(동/호수)] [자가여부(체크박스)] [상동여부(체크박스: 사업장과 동일)]

3. 사업자 정보 그룹 (Border Box)

Row 3-1 (한 줄에 배치): [상호명] [개업일(Date)]

Row 3-2 (한 줄에 배치): [업종(드롭다운)] [종목(Text)]

Row 3-3 (한 줄에 배치): [사업자등록번호] [재도전(드롭다운: 폐업,이전,변경)] [혁신(드롭다운: 배달앱,효율화,매출신장,기타)]

Row 3-4 (한 줄에 배치): [사업장소재지(도로명)] [상세주소] [자가여부(체크박스)]

Row 3-5 (한 줄에 배치): [최근매출] [Y-1매출] [Y-2매출] [Y-3매출] (단위: 억원 표시)

3. [섹션 2] 우측 상단: 문서 뷰어 (File Viewer)
헤더: [파일 업로드 버튼] 및 [업로드된 파일명 탭 리스트]

바디: 파일 내용을 보여주는 뷰어 영역 (파일이 없으면 '파일을 업로드하세요' 문구)

4. [섹션 3] 우측 하단(좌측): 상담 메모 (Memo)
UI: 카카오톡 채팅방 스타일.

기능: 상담사가 작성한 메모가 말풍선으로 쌓임. 하단에 입력창 + 전송 버튼.

5. [섹션 4] 우측 하단(우측): AI 질의 기능 (AI Chat)
UI: 상담 메모와 동일한 채팅 UI지만, 배경색을 약간 다르게 해서 구분.

용도: 시스템 로그나 AI 분석 결과가 출력되는 곳.

핵심: "섹션 1의 입력 필드들은 반드시 지정한 Row(행) 단위로 묶어서 한 줄에 배치해줘. 공간이 좁으면 flex-wrap을 쓰되 순서는 지켜야 해."
