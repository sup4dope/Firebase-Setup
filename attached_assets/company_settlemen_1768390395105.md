1. 개요 및 목적
•	목적: 실시간 매출 및 지출 데이터를 통합하여 경영진(Super Admin)에게 정확한 영업이익과 마케팅 효율(ROI) 지표를 제공함.
•	접근 권한: 오성경, 한지성 대표(super_admin) 계정만 접근 가능.
________________________________________
2. 주요 관리 항목 및 산출 공식
모든 수치는 '고객 입금일'을 기준으로 실시간 자동 계산됨.
항목	산출 공식 및 로직	비고
총매출 (Gross Revenue)	(전체 입금액 합계) - (환수 손실액)	입금일=계약일 기준
영업이익 (Operating Profit)	총매출 - (직원 수수료 + 마케팅비 + 고정비)	실질 순수익
세금 예비비 (Tax Reserve)	총매출 $\times$ 15%	경영 목표치 반영
DB 전환율 (CVR)	(확정 계약 건수 $\div$ 광고 유입 DB 수) $\times$ 100	실시간 카운팅
DB 효율성 (ROI)	(총매출 $\div$ 고정 광고비) $\times$ 100	CPT 효율 분석
________________________________________
3. 데이터 연동 및 입력 방식
A. 자동 연동 데이터
•	매출 및 수수료: '직원 정산 페이지'에서 데이터가 **확정(Approval)**되는 즉시 회사 정산 DB로 자동 합산됨.
•	마케팅 유입량: DB 테이블의 유입경로 필드값을 추적하여 '광고'로 유입된 DB 수를 실시간 카운팅함.
B. 수동/설정 데이터
•	마케팅 비용: 플랫폼별 월 고정 광고비(CPT)를 설정 메뉴에서 입력.
•	운영비(OPEX): 임대료(100만 원 이하), IT 도구 구독료(Replit, Vercel 등), 통신비 등.
•	기타 비용: 일시적 비용 발생 시 항목을 자유롭게 추가/삭제 가능하도록 구현.
________________________________________
4. 상세 기능 및 비즈니스 로직
① 지점 통합 관리
•	마케팅용 5개 지점(서울, 경인, 대전, 부산, 광주) 데이터를 별도로 분리하지 않고 하나의 통합 사업부로 합산하여 관리함.
② 환수(Clawback) 처리
•	환수 발생 시 해당 달의 총매출에서 마이너스 처리함.
•	단, 통계 및 관리를 위해 **'환수 손실'**이라는 별도 항목으로 금액과 사유를 기록함.
③ 세금 가상 적립
•	영업이익과 별개로 매출의 15%를 '세금 예비비'로 계산하여 대시보드에 시각화(게이지 또는 바 차트)함.
________________________________________
5. UI/UX 요구사항
•	대시보드 구성: 상단에 5대 지표(매출, 이익, 세금, ROI, 전환율) 배치.
•	보안: super_admin 권한이 없는 사용자가 URL로 직접 접근 시 403 Forbidden 에러 출력.
•	유연성: 운영비 리스트 하단에 [항목 추가] 버튼을 두어 실시간 비용 기록 대응.
