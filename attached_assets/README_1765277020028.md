# 💰 정책자금 컨설팅 CRM (Policy Fund Consulting CRM)

## 1. 프로젝트 개요
이 프로젝트는 정책자금 조달 자문 서비스를 위한 고객 관리 시스템입니다.
Firebase(Firestore)를 기반으로 고객의 상담부터 자금 집행까지의 상태(Funnel)를 관리하고, 영업일 기준의 정교한 KPI 예측 시스템을 제공합니다.

## 2. 기술 스택 (Tech Stack)
- **Frontend:** React, Vite, Tailwind CSS
- **Backend/DB:** Firebase (Firestore, Authentication)
- **Environment:** Replit

## 3. 핵심 기능 (Core Features)

### A. 사용자 권한 및 조직 관리 (ACL)
1. **권한 체계 (Role):**
   - **팀원 (Staff):** 본인 담당 고객만 조회/수정. 본인 예약/TO-DO 관리.
   - **팀장 (Team Leader):** 본인 팀 소속 직원의 데이터 조회. 팀 내 담당자 변경 가능. 팀원에게 TO-DO 할당 가능. (삭제 불가)
   - **총관리자 (Super Admin):** 모든 데이터 조회/수정/삭제. 팀 배정 및 공휴일 관리.
2. **팀 관리:** 팀명 변경 시 소속 직원/고객 데이터 일괄 동기화.

### B. 메인 대시보드 (Dashboard)
1. **퍼널(Funnel) UI:**
   - 상단 프로세스 흐름도 (상담대기 -> ... -> 집행완료).
   - 상태 변경은 리스트에서 고객 선택 후 버튼 클릭으로 처리.
   - 세부 상태(예: 0-1 단기부재)도 DB에 저장 및 통계 반영.
2. **고객 리스트 (Grid):**
   - **고유 ID:** `YYMMDD-XXX` 형식의 식별 코드 표시.
   - **보안 필드:** '수수료율' 등 민감 정보는 총관리자에게만 표시.
3. **KPI 위젯 (자동 계산):**
   - **계산 로직:** (현재 값 / 경과 영업일) * 월 전체 영업일
   - **영업일 기준:** 주말 및 '공휴일 설정'에 등록된 날짜 제외.
   - **예상 계약 건수:** 해당 월에 '계약완료' 상태를 1회 이상 거친 로그 기준.
   - **예상 매출액:** (승인 총액 / 경과 영업일) * 월 전체 영업일.

### C. 사이드바 (Sidebar)
1. **TO-DO List:**
   - 개인 업무 + 상급자가 할당한 업무 표시.
   - 필드: 내용, 마감기한(Due Date), 할당자(Assigner).
   - 팀 뷰: 팀장은 팀원들의 TO-DO를 보고 관리 가능.
2. **예약 대기:**
   - 본인 담당 고객의 예약 리스트.
   - 상급자는 하위 직원의 예약 현황 조회 가능.

### D. 관리자 기능 (Admin)
1. **공휴일 관리:** KPI 계산을 위한 휴무일 수동 등록/삭제.
2. **로그 추적:** 고객 상태 변경 시 [일시, 변경자, 이전상태, 이후상태] 기록.

## 4. 데이터베이스 설계 원칙
- 모든 데이터는 Firestore를 사용한다.
- 고객 삭제는 '논리적 삭제(숨김)'가 아닌 '물리적 삭제(완전 삭제)'로 구현하되 총관리자만 가능하다.
- 상태 변경 로그(`status_logs`)는 별도 컬렉션으로 분리하여 통계 속도를 확보한다.
