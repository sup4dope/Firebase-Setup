대시보드의 핵심인 **"고객 상세/등록 모달(Customer Detail Modal)"**을 구현할 차례야.
이 기능은 우리 CRM의 심장과도 같으니, 아래의 기획 내용을 빠짐없이 반영해서 코딩해줘.

### 1. 시스템 아키텍처 및 선행 작업
- **Firebase Storage 활성화:** 파일(PDF 등)을 업로드하기 위해 Firebase Storage 기능을 코드에 연결해줘.
- **주소 찾기:** 'Daum Postcode' 라이브러리(`react-daum-postcode`)를 사용하여 주소 검색 기능을 넣어줘.

### 2. 모달 레이아웃 (Layout)
- **크기:** 화면의 90%를 사용하는 대형 모달 (`w-11/12 max-w-7xl h-[90vh]`).
- **구조:** 좌측(입력 폼, 40%) vs 우측(탭 패널, 60%)의 2단 분할 구조.
- **헤더:** 고객명, 현재 상태(배지), [저장], [닫기] 버튼. (총관리자만 [삭제] 버튼 표시)

### 3. 좌측 패널: 데이터 입력 폼 (Scrollable)
다음 항목들을 그룹핑하여 입력받아줘.
- **[기본 정보]**
  - **고객명:** 텍스트
  - **사업자등록번호:** 텍스트 (필수, 중복 체크용 Key)
  - **상호명:** 텍스트
  - **연락처:** 텍스트
  - **설립일:** 날짜 선택 (Date Picker). **(중요: 선택 시 오늘 날짜 기준으로 '7년 초과 여부' 체크박스가 자동으로 켜지거나 꺼지도록 로직 구현)**
  - **7년 초과 여부:** 자동 계산되지만 수동 수정도 가능한 체크박스.
  - **주소:** [주소 검색] 버튼 + 기본주소 + 상세주소 입력란.
- **[재무 정보]**
  - **신용점수:** 숫자
  - **작년 매출:** 숫자 (단위: 억원)
  - **3년 평균 매출:** 숫자 (단위: 억원)
- **[관리 정보]**
  - **담당자:** 드롭다운 (현재 등록된 유저 리스트 불러오기)
  - **진행기관:** 드롭다운
  - **수수료율(%):** 숫자 (⚠️ **로그인한 유저가 '총관리자'일 때만 보이게 조건부 렌더링**)

### 4. 우측 패널: 4개의 탭 (Tabs) 구성
우측 패널 상단에 탭 메뉴를 만들고, 클릭 시 내용이 바뀌게 해줘.

- **Tab 1: 상담 메모 (Chat)**
  - 카카오톡 스타일의 메모장.
  - 작성자, 시간, 내용 말풍선 표시.
  - 하단에 입력창 + [등록] 버튼.
- **Tab 2: 문서 관리 (Documents & PDF Viewer) ** (★중요)
  - **기능:** 사업계획서, 재무제표 등 PDF 파일을 업로드하고 볼 수 있는 공간.
  - **레이아웃:**
    - 상단: [파일 업로드] 버튼 + 파일 리스트 (파일명, 업로드 날짜, 삭제버튼).
    - 하단: 리스트에서 파일을 클릭하면, 해당 PDF를 바로 볼 수 있는 **미리보기(Preview) 영역** (`iframe` 또는 `object` 태그 활용).
- **Tab 3: 변경 이력 (History)**
  - 상태 변경, 담당자 변경 등의 로그를 타임라인으로 표시.
- **Tab 4: 할 일 (TO-DO)**
  - 이 고객과 연동된 할 일 목록 조회 및 추가.

### 5. 데이터 처리 로직
- **신규 등록:** [+ 고객 추가] 버튼으로 진입 시 빈 폼 노출. 저장 시 `YYYYMMDD-Sequence` 형태의 Readable ID 생성.
- **수정:** 리스트에서 클릭 시 데이터 바인딩. 수정 후 저장 시 Update.

요약: "좌측은 스마트한 입력 폼(7년 자동계산), 우측은 채팅 메모와 PDF 뷰어가 포함된 탭 메뉴를 가진 완벽한 상세 팝업을 만들어줘."
