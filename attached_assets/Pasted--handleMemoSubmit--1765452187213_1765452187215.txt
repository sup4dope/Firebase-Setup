사용자가 "내부에서 메모를 저장했는데 외부에서 보면 없고, 내부에서도 사라졌다가 다시 입력하면 나타난다"는 기이한 현상을 발견했어.
원인은 `handleMemoSubmit`에서 `memoHistory` 상태(State) 전체를 덮어쓰기(`set`) 방식으로 저장하다 보니, **초기 로딩이 덜 된 상태에서 저장하면 기존 내역을 날려먹거나 누락**하기 때문이야.

`CustomerDetailModal.tsx`의 `handleMemoSubmit` 함수를 수정해서, `memo_history` 필드를 업데이트할 때 **Firestore의 `arrayUnion` 기능**을 사용하도록 바꿔줘.

### 1. 상단 Import 추가
* `firebase/firestore`에서 `arrayUnion`을 불러와야 해.
```typescript
import { 
  // ... 기존 import들 ...
  arrayUnion 
} from "firebase/firestore";
2. handleMemoSubmit 로직 수정 (안전한 추가)
핵심: customers 문서를 업데이트할 때, 배열을 통째로 교체하지 말고 **arrayUnion(newLog)**를 사용해서 기존 데이터 끝에 안전하게 붙여넣어.

TypeScript

  const handleMemoSubmit = async () => {
    const content = memoInput.trim();
    if (!content) return;

    // 1. 새 로그 객체 생성
    const newLog = {
      id: Date.now().toString(),
      content: content,
      author_name: currentUser?.displayName || "관리자",
      created_at: new Date(),
      type: "memo"
    };

    // 2. 화면 선반영 (UI용)
    setMemoHistory(prev => [...prev, newLog]);
    setMemoInput(""); 

    try {
      // 3. [로그 컬렉션] 저장 (내부용)
      await addDoc(collection(db, "counseling_logs"), {
        customer_id: customerId,
        content: content,
        author_name: currentUser?.displayName || "관리자",
        created_at: new Date(),
        type: "memo"
      });

      // 4. [고객 문서] 안전한 업데이트 (외부용)
      if (customerId) {
        // ★핵심: 기존 배열을 몰라도 됨. 그냥 끝에 추가해라! (arrayUnion)
        // cleanData로 날짜 에러 방지 후 arrayUnion 사용
        const cleanLog = cleanData(newLog); 

        await updateDoc(doc(db, "customers", customerId), {
          recent_memo: content,
          latest_memo: content,
          last_memo_date: new Date(),
          // ★ 여기서 배열 전체를 덮어쓰지 않고 추가만 함 (덮어쓰기 사고 방지)
          memo_history: arrayUnion(cleanLog)
        });
        
        // 5. 로컬 formData도 동기화 (팀킬 방지)
        // 로컬에서는 arrayUnion을 못 쓰니 수동으로 추가
        setFormData(prev => ({ 
          ...prev, 
          recent_memo: content,
          last_memo_date: new Date(),
          memo_history: [...(prev.memo_history || []), newLog]
        }));
      }

      // 6. 부모 갱신 알림
      if (onSave) {
        onSave({ 
          id: customerId, 
          recent_memo: content,
          last_memo_date: new Date()
        });
      }

    } catch (error) {
      console.error("메모 저장 실패:", error);
    }
  };
요약: "기존 데이터를 덮어쓰는 위험한 방식 대신, arrayUnion을 써서 무조건 끝에 추가되게 해. 그럼 로딩이 덜 됐어도 기존 데이터가 안 날아가."
