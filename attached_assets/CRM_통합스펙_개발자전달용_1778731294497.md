# MSGY CRM × AI팀 자동기입 통합 스펙
**대상:** CRM 개발자  
**버전:** v2.0 (2026-05-14)  
**목적:** 대표가 사용하는 로컬 AI 보조 시스템(크롬 확장프로그램)이 CRM 상세페이지에서 고객 정보를 자동 캡처하여, 외부 사이트(홈택스 간편인증 등)에 자동으로 붙여넣을 수 있도록 합니다.

---

## ⚡ TL;DR (요약)

CRM 측이 할 일은 **딱 두 가지** 중 하나입니다:

### 옵션 A: 작업 없음 (자동 스크래핑) — 권장
- 현재 CRM이 라벨 텍스트("이름", "주민번호(앞)" 등) + 인접 input 구조를 따르므로 **변경 불필요**.
- 확장프로그램이 라벨 기반으로 자동 추출합니다.
- 단점: CRM HTML 구조가 크게 바뀌면 자동 인식 실패 가능.

### 옵션 B: 명시적 마크업 추가 (선택) — 안정성 ↑
- 상세 모달 컨테이너에 `id="ai-team-data"` 와 `data-*` 속성 추가.
- HTML 구조 변경에 영향받지 않음.

**현재 CRM은 옵션 A로 동작 가능합니다.** 그러나 안정성과 유지보수를 위해 옵션 B를 권장합니다.

## 📐 추출 대상 필드

확장프로그램이 인식하는 라벨 텍스트(옵션 A) / data-* 속성(옵션 B):

| 필드 | 라벨 텍스트 (옵션 A) | data-* 속성 (옵션 B) | 변환 규칙 |
|------|-----------------|--------------------|---------|
| 이름 | `이름`, `성명`, `고객명`, `대표자` | `data-ceo-name` | 그대로 |
| 주민번호 앞 | `주민번호(앞)`, `주민번호 앞`, `주민등록번호` | (옵션 B에선 생년월일 직접) | 6자리 |
| 주민번호 뒤 | `주민번호(뒤)`, `주민번호 뒤` | - | 첫자리만 사용 |
| 생년월일 | `생년월일`, `생일` | `data-ceo-birthdate` | YYYYMMDD 8자리 |
| 휴대폰 | `연락처` (3분할 input), `휴대폰`, `전화` | `data-ceo-phone` | 01012345678 11자리 |
| 상호명 | `상호명`, `기업명`, `회사명` | `data-company-name` | 그대로 |
| 사업자번호 | `사업자번호` | `data-biz-number` | 숫자만 |
| 업종 | `업종` | `data-industry` | 그대로 |
| 사업장 소재지 | `사업장 소재지`, `소재지`, `주소` | `data-region` | 그대로 |
| 신용점수 | `신용점수` | `data-credit-score` | 숫자 |

### 주민번호 → 생년월일 자동 변환

확장이 자동으로 처리합니다:
- 앞 6자리 = `YYMMDD`
- 뒤 첫자리:
  - `1,2,5,6` → 1900년대 (예: `750627-2` → `19750627`)
  - `3,4,7,8` → 2000년대 (예: `050627-4` → `20050627`)
  - `9,0` → 1800년대

```html

    <input value="690">
    <label>주민번호(앞)</label>
    <input value="750627">
    <label>주민번호(뒤)</label>
    <input value="2336913">
    ...
  </div>
</div>
```

### 옵션 B 적용 후 (최소 변경)

**방법 1: 컨테이너에 속성 추가 (가장 간단)**

```html
<!-- 변경 후: id 와 data-* 만 추가 -->
<div class="customer-modal"
     id="ai-team-data"
     data-ceo-name="홍주영"
     data-ceo-birthdate="19750627"
     data-ceo-phone="01056943000"
     data-company-name="로또기업"
     data-biz-number="4783600430"
     data-industry="소매업"
     data-region="대전광역시 동구 동서대로 1684(성남동)"
     data-credit-score="690">
  <!-- 기존 내용 유지 -->
  ...
</div>
```

서버에서 모달을 렌더링할 때 데이터를 속성으로 한 번 더 내려주면 됩니다.

**방법 2: 자식 요소에 data-field 추가 (구조가 분산된 경우)**

```html
<div id="ai-team-data" data-company-name="로또기업">
  <input data-field="ceo_name" value="홍주영">
  <input data-field="ceo_birthdate" value="19750627">
  <input data-field="ceo_phone" value="01056943000">
  <input data-field="biz_number" value="4783600430">
</div>
```

## ✅ CRM 측 작업 체크리스트

### 옵션 A (변경 없이 진행)
- [ ] 라벨 텍스트가 표준에 맞는지 확인:
  - 이름 → `이름` (○) / `이 름` (○, 공백 무시)
  - 주민번호 앞 → `주민번호(앞)` 또는 `주민번호 앞` (○)
  - 연락처 → `연락처` (3분할 input 구조)
- [ ] 라벨과 input 이 같은 부모 영역 안에 있어야 함

### 옵션 B (마크업 추가 — 권장)
- [ ] 상세 모달 최상위 컨테이너에 `id="ai-team-data"` 추가
- [ ] 다음 `data-*` 속성 추가 (값은 DB 값 그대로):
  - `data-ceo-name`
  - `data-ceo-birthdate` (YYYYMMDD)
  - `data-ceo-phone` (01012345678)
  - `data-company-name`
  - `data-biz-number`
  - (선택) `data-industry`, `data-region`, `data-credit-score`
- [ ] 모달이 SPA(React/Vue 등)로 동적 렌더링되어도 OK (MutationObserver로 감지함)

## 🧪 테스트 방법

### CRM 측에서 검증

확장 미설치 상태에서도 검증 가능:

```javascript
// 브라우저 콘솔에서 실행
const data = document.querySelector('[data-ai-team-crm="true"], #ai-team-data');
if (data) {
  console.log("✅ 컨테이너 발견:", data);
  console.log("data-* 속성:");
  for (const attr of data.attributes) {
    if (attr.name.startsWith("data-")) console.log(`  ${attr.name} = ${attr.value}`);
  }
} else {
  console.log("❌ 마크업 누락: id='ai-team-data' 컨테이너를 찾을 수 없음");
}
```

기대 결과:
```
✅ 컨테이너 발견: <div id="ai-team-data" ...>
data-* 속성:
  data-ceo-name = 홍주영
  data-ceo-birthdate = 19750627
  data-ceo-phone = 01056943000
  ...
```

### 통합 테스트

1. 대표 PC에 확장 설치 완료 확인
2. CRM 상세페이지 열기 → 우상단 토스트 `✅ 홍주영 정보 캡처됨` 확인
3. 새 탭에서 https://www.hometax.go.kr → 간편인증 모달 진입
4. 우측 하단 플로팅 패널 표시 확인
5. 드롭다운에서 `홍주영` 선택 → `붙여넣기` 클릭
6. 이름 / 생년월일 / 휴대폰 필드 자동 기입 확인

## ❓ FAQ

**Q1. CRM이 HTTPS인데 로컬봇이 HTTP라 차단되지 않나요?**  
A. 차단됩니다. 그래서 **크롬 확장프로그램**을 중간 다리로 사용합니다. 확장은 manifest에 권한이 있어 HTTPS 페이지에서 localhost로 fetch 가능합니다.

**Q2. 모달이 React/Vue 같은 SPA로 렌더링되는데 괜찮나요?**  
A. 괜찮습니다. 확장이 `MutationObserver`로 DOM 변화를 감지합니다. 모달이 열릴 때마다 자동 캡처됩니다.

**Q3. data-* 속성을 서버 사이드에서 못 추가하면 어떻게 하나요?**  
A. 클라이언트 JS로 추가해도 됩니다:
```javascript
// 모달 열릴 때 실행
function applyAITeamMarkup(modal, customer) {
  modal.id = "ai-team-data";
  modal.dataset.ceoName     = customer.name;
  modal.dataset.ceoBirthdate = customer.birthdate;
  modal.dataset.ceoPhone    = customer.phone;
  // ...
}
```

**Q4. 옵션 A 자동 스크래핑이 실패할 가능성은?**  
A. 라벨 텍스트가 표준에서 벗어나거나 (`고객 성명`, `연락처번호` 등) 라벨과 input이 분리된 구조면 실패할 수 있습니다. 그래서 옵션 B를 권장합니다.

**Q5. 확장 권한이 너무 광범위한 것 같은데(`<all_urls>`)?**  
A. 간편인증을 어느 사이트에서 쓸지 모르므로 모든 도메인에 매칭해야 합니다. 단, 실제 활동은:
- CRM 페이지 → 캡처
- 간편인증 페이지 (이름·생년월일·휴대폰 필드 존재) → 패널 표시
- 그 외 → 아무 동작 안 함

**Q6. 다른 직원도 사용하나요?**  
A. 현재는 대표 1인 사용 기준. 직원에게도 배포하려면 확장 설치 + 텔레그램 봇 토큰 공유 필요. (Phase 3에서 검토)
